Eslint与Prettier

为什么要使用eslint+prettier。

目的是为了规范代码，统一风格，便于后续人员维护。

在项目生成的时候，关于代码格式化的选择：

? Pick a linter / formatter config:
  TSLint
  ESLint with error prevention only
  ESLint + Airbnb config
  ESLint + Standard config
> ESLint + Prettier

lint选择ESLint + Prettier，配置方式选择In dedicated config files，根目录下会自动生成一个.eslintrc.js文件。

module.exports = {
  root: true,
  env: {
    node: true
  },
  extends: ["plugin:vue/essential", "@vue/prettier", "@vue/typescript"],
  rules: {
    "no-console": process.env.NODE_ENV === "production" ? "error" : "off",
    "no-debugger": process.env.NODE_ENV === "production" ? "error" : "off"
  },
  parserOptions: {
    parser: "@typescript-eslint/parser"
  },
  overrides: [
    {
      files: ["**/__tests__/*.{j,t}s?(x)"],
      env: {
        mocha: true
      }
    }
  ]
};

extends：plugin:vue/essential作用是加载eslint-plugin-vue插件，默认启用essential级别的规制，该插件可在node_modules中找下，如果没有的话，需要手动添加。

eslint-plugin-vue把所有规则分为四个类别，依次为：base, essential, strongly-recommended, recommended，后面的每个类别都是对前一个类别的拓展。除了这四个类别外，还有两个未归类的规则，所有的类别及规则都可以在这里查看。
